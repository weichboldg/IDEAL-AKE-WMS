**Hintergrund:** Diese 4 Services brauchen alle eine `IConfiguration`-basierte DB-Verbindung (raw ADO.NET). Success-Path-Tests gegen InMemory-DB sind nicht moeglich. Daher: **Failure-Path-Lifecycle-Tests** — analog zu `PartRequisitionEmailServiceTests` aus v1.15.1 (Pattern in `IDEALAKEWMSService.Tests/Services/PartRequisitionEmailServiceTests.cs`).

Die Tests verifizieren, dass:
1. `BeginRunAsync` aufgerufen wird (run wird im `FakeSyncLogger.Runs` registriert mit korrektem ServiceName)
2. `FinishFailedAsync` aufgerufen wird (`FinishedFailed == true`)
3. Die exception wird re-thrown

using Microsoft.Extensions.Logging.Abstractions;

namespace IDEALAKEWMSService.Tests.Services;

public class OseonSyncServiceTests

    {
        var config = new ConfigurationBuilder().Build();
        return new OseonSyncService(config, NullLogger<OseonSyncService>.Instance, fakeLogger);
    }

    [Fact]
    public async Task SyncProductionOrders_writes_lifecycle_to_synclogger_via_failure_path()
