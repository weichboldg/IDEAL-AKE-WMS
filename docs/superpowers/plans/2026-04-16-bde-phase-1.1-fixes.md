## Task 1: Terminal — Scan-Button + Heuristik-Fix

**Files:**
- Modify: `IdealAkeWms/wwwroot/js/bde-terminal.js`
- Modify: `IdealAkeWms/Views/BdeTerminal/Index.cshtml`

### Problem

1. Kein visueller "Scannen"-Button — nur Enter-Taste als Trigger (unbrauchbar ohne physischen Scanner)
2. FA-Nummern werden fälschlich als Personalnummer interpretiert wenn sie kein Komma/Slash enthalten

### Lösung

**Scan-Button:** Neben dem Input-Feld einen `<button id="btnScan">Scannen</button>` einfügen. Click-Handler ruft dieselbe `handleScan(raw)` auf wie Enter.

**Heuristik-Fix:** Statt einer fragilen Regex zwei klare Eingabe-Modi:

```javascript
// Zwei separate Inputs statt einer Heuristik:
// Mode 1: "Operator" — Personalnummer-Feld (oben, immer sichtbar)
// Mode 2: "FA/AG" — getrenntes Feld oder der AG-Button-Klick (Finding 3)
```

**Alternativ (einfacher):** Ein einzelnes Scan-Feld beibehalten, aber die Heuristik verbessern:
- Personalnummern sind typischerweise rein numerisch oder haben ein festes Prefix (z.B. "P" + Ziffern)
- FA-Nummern enthalten typischerweise Buchstaben UND Ziffern (z.B. "WA-12345" oder "FA-12345")
- Trennzeichen `,` oder `/` nach FA-Nummer → AG-Nummer

**Gewählter Ansatz: Zwei separate Scan-Bereiche.**
- Oben: "Operator scannen" — Input + Button, akzeptiert NUR Personalnummer
- Darunter: "Arbeitsgang scannen" — Input + Button, akzeptiert FA-Nummer,AG-Nummer Format
- Plus die AG-Buttons aus Finding 3 als dritte Eingabe-Möglichkeit

### Neuer API-Endpoint

```csharp
[HttpGet("available-operations/{workplaceId:int}")]
public async Task<IActionResult> GetAvailableOperations(int workplaceId)
{
    // 1) Offene WorkOperations an dieser Werkbank (nicht erledigt, nicht in aktiver Buchung)
    var workOps = await _workOps.GetOpenByWorkplaceIdAsync(workplaceId);
    // Filter: nur WorkOperations die NICHT bereits eine aktive (EndedAt=NULL, !IsCancelled) Buchung haben
    var activeWoIds = (await _bookings.GetActiveCockpitAsync())
        .Where(b => b.WorkOperationId.HasValue)
        .Select(b => b.WorkOperationId!.Value)
        .ToHashSet();

    var productive = workOps
        .Where(wo => !activeWoIds.Contains(wo.Id))
        .Select(wo => new {
            id = wo.Id,
            label = $"{wo.ProductionOrder.OrderNumber} / {wo.OperationNumber} — {wo.Name}",
            orderNumber = wo.ProductionOrder.OrderNumber,
            operationNumber = wo.OperationNumber,
            type = "productive"
        });

    // 2) Ungeplante Aktivitäten
    var activities = (await _activities.GetAllActiveAsync())
        .Select(a => new {
            id = a.Id,
            label = $"{a.Code} — {a.Name}",
            orderNumber = (string?)null,
            operationNumber = (string?)null,
            type = "unplanned"
        });

    return Ok(new { productive, unplanned = activities });
}
```

### Terminal-JS: AG-Buttons laden und rendern

```javascript
async function loadAvailableOperations() {
    const r = await fetch(`/api/bde/available-operations/${workplaceId}`);
    if (!r.ok) return;
    const data = await r.json();
    renderOperationButtons(data.productive, data.unplanned);
}

function renderOperationButtons(productive, unplanned) {
    const container = document.getElementById('operationButtons');
    let html = '';

    if (productive.length > 0) {
        html += '<h6 class="mt-2 text-muted">Produktive Arbeitsgänge</h6>';
        html += '<div class="d-flex flex-wrap gap-2 mb-3">';
        productive.forEach(op => {
            html += `<button class="btn btn-outline-success btn-lg bde-op-btn" data-wo-id="${op.id}" data-type="productive">${op.label}</button>`;
        });
        html += '</div>';
    }

    if (unplanned.length > 0) {
        html += '<h6 class="text-muted">Ungeplante Tätigkeiten</h6>';
        html += '<div class="d-flex flex-wrap gap-2 mb-3">';
        unplanned.forEach(a => {
            html += `<button class="btn btn-outline-secondary btn-lg bde-op-btn" data-activity-id="${a.id}" data-type="unplanned">${a.label}</button>`;
        });
        html += '</div>';
    }

    if (!productive.length && !unplanned.length) {
        html = '<p class="text-muted">Keine offenen Arbeitsgänge an dieser Werkbank.</p>';
    }

    container.innerHTML = html;
    bindOperationButtonHandlers();
}

function bindOperationButtonHandlers() {
    document.querySelectorAll('.bde-op-btn').forEach(btn => {
        btn.addEventListener('click', async () => {
            if (btn.dataset.type === 'productive') {
                currentWorkOp = { id: parseInt(btn.dataset.woId) };
                // Zeige Start-Buttons (Rüsten / Produktion)
                renderStartButtons();
            } else {
                // Ungeplante Tätigkeit direkt starten
                await post('/BdeTerminal/StartActivity', {
                    operatorId: currentOperator.id,
                    activityId: parseInt(btn.dataset.activityId),
                    workplaceId, terminalId
                });
                await renderState();
                await loadAvailableOperations();
            }
        });
    });
}
```

### JS-Änderung (Kachel-Rendering)

```javascript
function render(data) {
    grid.innerHTML = data.workplaces.map(t => {
        if (t.status === 'Idle') {
            return `<div class="col-md-4 col-lg-3"><div class="card text-white bg-secondary">
                <div class="card-body"><h5>${t.workplaceName}</h5><em>Frei</em></div></div></div>`;
        }
        const bookingHtml = t.bookings.map(b => {
            const target = b.orderNumber
                ? `${b.orderNumber} / ${b.operationNumber} — ${b.operationName}`
                : b.activityName;
            const color = colorForBooking(b);
            return `<div class="cockpit-booking ${color} p-2 rounded mb-1">
                <strong>${b.bookingType}</strong> · ${b.operatorName}
                <div class="small">${target ?? ''}</div>
                <div class="small">Seit: ${fmt(b.runtimeSeconds)}</div>
            </div>`;
        }).join('');
        return `<div class="col-md-4 col-lg-3"><div class="card">
            <div class="card-header"><h5 class="mb-0">${t.workplaceName}</h5></div>
            <div class="card-body p-2">${bookingHtml}</div>
        </div></div>`;
    }).join('');
    serverTimeEl.textContent = new Date(data.serverTime).toLocaleTimeString();
}

function colorForBooking(b) {
    switch (b.bookingType) {
        case 'Setup': return 'bg-orange';
        case 'Production': return 'bg-success text-white';
        case 'Activity': return 'bg-info text-white';
        default: return 'bg-secondary text-white';
    }
}
```

Die aktuelle BDE-Hilfe ist zu knapp. Ersetzen durch eine konkrete Schritt-für-Schritt-Anleitung:

### Inhalt

```
## Betriebsdatenerfassung (BDE)

### Rollen
- **BDE-Mitarbeiter** (bde_user): Terminal-Buchung — Arbeitsgänge scannen, Status wechseln, Mengen melden
- **BDE-Schichtleiter** (bde_shiftlead): + Stammdaten pflegen, Buchungsliste, Cockpit
- **BDE-Admin** (bde_admin): + Buchungen korrigieren/stornieren, Terminals konfigurieren

### Terminal-Bedienung (Schritt für Schritt)

1. **Operator anmelden:** Personalnummer am Terminal scannen oder eingeben + "Scannen" klicken. Der Name wird als Badge angezeigt.
2. **Arbeitsgang wählen:** Nach der Anmeldung erscheinen die verfügbaren Arbeitsgänge als Buttons, gruppiert nach produktiven (aus FA) und ungeplanten Tätigkeiten (Wartung, Reinigung, …). Alternativ: FA-Nummer und AG-Nummer vom Laufzettel scannen.
3. **Rüsten starten:** Für einen produktiven AG → "Rüsten starten" klicken. Rüstzeit wird erfasst.
4. **Produktion starten:** Rüsten abgeschlossen → "Produktion starten" (schließt Rüstung automatisch).
5. **Teilmenge melden:** Während laufender Produktion → "Teilfertigmeldung" → Gutmenge + Ausschuss eingeben.
6. **Pausieren:** "Pause" klicken → bei Produktion optional Mengen eingeben. Ein anderer Operator kann den AG fortsetzen.
7. **Beenden:** "Beenden" → finale Gutmenge + Ausschuss eingeben. AG ist abgeschlossen.
8. **Operator wechseln:** "Wechseln"-Button im Operator-Badge klicken → neuen Operator scannen.

### Cockpit (Schichtleiter/Admin)
- Zeigt alle Werkbänke in Echtzeit (5-Sekunden-Aktualisierung)
- Farbcodierung: Grün = Produktion, Orange = Rüsten, Blau = Ungeplant, Grau = Frei
- Mehrere Operatoren pro Werkbank werden einzeln angezeigt

### Buchungsübersicht (Schichtleiter/Admin)
- Filtert standardmäßig auf heute, mit Spaltenfiltern für Operator, Werkbank, Typ, Status
- Admins können Buchungen bearbeiten, stornieren oder vergessene Buchungen manuell schließen

### Stammdaten (Schichtleiter/Admin)
- **Operatoren:** Werker mit Personalnummer anlegen/bearbeiten (optional mit WMS-Benutzer verknüpfen)
- **Aktivitäten:** Ungeplante Kategorien wie "Wartung", "Reinigung" pflegen
- **Terminals:** Welcher WMS-Benutzer an welcher Werkbank angemeldet ist (nur Admin)
```
